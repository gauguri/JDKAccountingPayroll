Fix: expense notes column, tax-table seeding in tests, startup & deps

Bug fixes found by running the test suite on a clean machine. One is a real production bug (expense creation was broken); the rest are test-environment and housekeeping fixes. All 26 backend tests now pass.

🐞 Production bug: the expense endpoint passed notes=… but the Expense model had no notes column, so every expense save (and the reports/CPA flows that create expenses) raised a 500 in the live app. Fixed by adding the column.

What changed

  • Add Expense.notes column (models/transactions.py) — fixes expense creation, expense listing, and the CPA export that reads notes.
  • Seed sample tax tables in tests (tests/conftest.py) — the startup seeder only runs under a live server, so the test client found no rates and payroll taxes computed as 0.00. Seeding in the fixture fixes the payroll tests (now SS $99.20, Medicare $23.20 on a $1,600 check).
  • Authenticate the /tax-rules tests (tests/test_payroll.py) — they were calling a protected endpoint without a session.
  • Migrate startup to FastAPI lifespan (app/main.py) — replaces the deprecated on_event("startup").
  • Refactor a payroll test helper so no test function returns a value (future pytest treats that as an error).
  • Relax the psycopg pin (requirements.txt: psycopg[binary]>=3.2,<4) so installs work on newer Python versions.

Notes for reviewers

  • No schema migration tooling yet (MVP stage) — tables are created from models at startup, so the new notes column appears automatically on a fresh DB. An existing Postgres volume would need the column added or the dev volume recreated.
  • No behavior change to the payroll math — the earlier 0.00 values were purely missing seed data in tests, not a calculation error.
