MVP 1 — Daily bookkeeping

First working version of JDK Books: a simple, senior-friendly QuickBooks alternative for the two shops (apparel/embroidery and custom signs). Records income and expenses, explains profit, and packages everything for the CPA. No payroll or tax filing yet (those are MVP 2+).

base: main compare: mvp1-bookkeeping

What's included

  • Auth & multi-company — email/password login; both businesses run as separate sets of books (his C-corp, her LLC), New Jersey defaults, append-only audit log on every change.
  • Chart of accounts — preloaded in plain English for an apparel/embroidery/signs shop (Blank Shirts/Hats, Vinyl/Sign Materials, Sales Tax Payable, etc.). Built-in accounts can be hidden but not deleted; accounts with transactions are protected.
  • Income & expense entry — wizard with an explain-before-post preview so nothing hits the books until the user confirms. NJ sales tax booked as a liability; credit-card purchases tracked correctly.
  • Bank CSV import — column mapping (remembered per bank), duplicate detection, category suggestions, "needs review" flag.
  • Reports — Profit & Loss, Balance Sheet, Trial Balance, General Ledger, each downloadable as PDF or CSV.
  • CPA export — one-click ZIP with all reports plus a notes file listing items flagged for CPA review.
  • QuickBooks import — import module with a working CSV-export path; live QBO OAuth is wired at deploy time with Intuit app credentials.
  • Frontend — React + TypeScript + Tailwind, large fonts, big buttons, "What do you want to do today?" home screen.

Stack

React/TS/Tailwind · FastAPI · PostgreSQL (SQLite for tests/local) · Docker Compose.

How to test

# Full stack
docker compose up --build      # web :3000, api :8000/docs

# Backend tests
cd backend && pip install -r requirements.txt && pytest

Tests cover auth + multi-company isolation, chart-of-accounts guards, income/expense posting with NJ sales tax, double-entry tie-outs (trial balance balances; Assets = Liabilities + Equity), CSV import with dedupe, report exports, and the CPA ZIP.

Design guardrails honored

Explain-before-post · AI-suggests-only (none wired in yet) · tax handling as data, not hardcoded · "organize, don't file."

Out of scope (follow-ups)

Receipt upload + OCR, optional MFA, backup/restore UI, and bank reconciliation (Phase 6). Payroll is MVP 2.

Notes for reviewers

  • New users register via the web app (creates the first company); add the second business with "+ Add business."
  • Books use proper double-entry; corrections post reversing entries rather than editing history.
